Notes on explanation...
The StegoLock Document Retrieval and Decryption Flow (*payts na ang "Document Retrieval" as a label for this process) is a structured process designed (*"structured process designed" - murag exaggerated ra kaayog impression)
to securely recover (*better change to "retrieve" to stay consistent with the use of term.. walay "recover" na word gigamit sa diagram.. and "recover" implies na nadamage ang file maong need i-recover) information from cloud storage.
The process is initiated when a user interacts with the web application by clicking the "Decrypt" button, signaling a request for document retrieval. 
Following this trigger, the system accesses cloud storage to fetch the Segmented Covers (*never nato gi-segment ang cover along the process.. however, the cover contains the segmented encryption data, which ang ang ciphertext... "Segmented Covers" implies na gi-split nato ang cover, which is wala tay ani nga step in any system process),
which act as the carrier files (*act as the stego or carrier files) for the hidden information. Once these covers are obtained, the system performs a
specialized extraction process (*exajj rasad pag use sa term nga "specialized") to isolate the Encryption Data , The segmented ciphertext that was hidden (*kung gusto mag gamit ug more technical-sounding na term for hiding, use "embedding" "embedded" basta "embed"...) within each individual cover during the initial upload phase.
After extraction, the system moves into a critical reconstruction phase (*pwede ra tanggalon ang "critical", exajj rasad) where it performs an Assembly of Segmented Ciphertext.
This step is vital because the data is stored in fragments; the system must reorder and join these segments correctly to rebuild the complete encrypted file.
Once the ciphertext is fully assembled, it is processed through the AES (*AES-GCM) algorithm using a managed lightweight key (*remove "using a managed lightweight key", the use of this phrase brings some kind of confusionsa reader.. kung gusto jud ni i-add, rephrase.. kung di makaya, tanggala).
This decryption step transforms the secured data back into its readable form (*this can be omitted, this is common jargon, no need ma-emphasize.. or just make it one with the previous sentence, no need iseparate, kay ang impression sa reader kung iseparate ni in a sentence, gina emphasize nato nga "decryption process diay to sya guys" which is not necessary).
The flow concludes (*the process ends) with the Original Document being successfully retrieved and presented to the user in its initial, unencrypted state.
